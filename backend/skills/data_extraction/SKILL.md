# Data Extraction 技能使用指南

## 技能概述

本技能专注于从非结构化文本中提取结构化数据，包括：
- 命名实体识别（人名、地名、机构名等）
- 时间日期提取和标准化
- 数据模式识别和提取
- 数据验证和清洗

## 使用方法

### 实体识别

在抓取的文本中识别：
1. **人名**：职务姓名、作者姓名等
2. **机构**：政府部门、公司名称等
3. **地点**：省市、地址等
4. **时间**：日期、时间段、任期等

### 数据结构化

将提取的信息组织为结构化格式：
```json
{
  "name": "姓名",
  "title": "职务",
  "organization": "机构",
  "start_date": "开始日期",
  "end_date": "结束日期",
  "source": "数据源"
}
```

## 提取规则

### 1. 职务信息提取

- **职务全名**：优先提取完整职务名称
- **职务层级**：区分正职、副职、助理等
- **部门**：提取所属部门或机构

### 2. 时间信息提取

- **标准格式**：YYYY-MM-DD, YYYY年MM月DD日
- **相对时间**："2023年至今" → start_date: "2023-01-01"
- **模糊时间**："近三年" → 需结合上下文推断

### 3. 数据验证

- **必填字段**：姓名、职务至少一个
- **格式检查**：日期格式、电话号码等
- **数据一致性**：检查逻辑矛盾（如结束时间早于开始时间）

## 使用流程

1. 获取原始文本内容
2. 使用正则表达式或 NLP 技术提取实体
3. 标准化数据格式
4. 验证数据完整性
5. 保存为结构化格式（JSON/CSV/Excel）

## 最佳实践

- 优先使用工具提供的提取函数
- 人工验证提取结果
- 保留原始数据用于追溯
